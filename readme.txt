Instructions for getting started:
    The Command Line Interface was made using python so installing python is required the version that we used is Python 3.13.9.
    There are 3 SQL related files
	1. database.sql - this contains the table creation and the populated tables.
	2. testQ.sql - this contains the test queries, see below on how to execute.
	3. database.db - this contains the compiled database.sql file
    There are 2 python scripts separate from the CLI:
        1. setup.py:
            setup.py is used to compile the database.sql file into database.db which contains the populated tables from database.sql
            If there is no databse.db file it can be generated by running the command "python setup.py"
        2. runTests.py
            runTests.py was created to execute the test queries. This script can be be ran by using command "python runTests.py" this should print all the outputs in the terminal.
            runTests can also be called within the CLI by typing 6 on the main page and it will output the test query results whithin the CLI.
            Instead of using runTests.py script the test queries can be executed by running the command "sqlite3 database.db < test_queries.sql" in the terminal but this requires sqlite3 to be installed.
        
    To run the CLI we suggest that you run it by double clicking the app.py file which should pop up a command prompt (Since it uses a command prompt this will not work on devices other than windows) you may be able to run it by typing the command "python app.py" into the terminal but this was causing problems.

    Once you have the app.py running you will be prompted with the message:
        What Action would you like to perform?
        1. Add
        2. Delete
        3. Edit
        4. View
        5. Clear
        6. Run Test Queries
        7. Exit

        Enter choice (1-7):

    Where you tell the program what types of commands you would like to perform.
    Once Add, Delete, Edit, or View is selected the user will be prompted with more questions reguarding the type of action they are performing.
    It is up to user to answer the following prompts according to the action they wish to perform.  
    Once all fields of an action has been completed you should get a status on whether your command was successful or not.
    Clear, Run Test Queries, and Exit are all single action commands in wich Clear clears the terminal, Run Test Queries prints the test query results and Exit ends the program.

    The Add Command is similar to running the SQL command INSERT INTO
    The Edit command will first SELECT * FROM "table" WHERE "id" and then will use the SQL command UPDATE SET based on the id given and the requested fields to change.
    The Delete command will first SELECT * FROM "table" WHERE "id" and then will use the SQL command DELETE FROM "table" WHERE "id".
    The View command is similar to the SQL command SELECT * FROM and then will loop and print each item in that table.
        There are also 2 extra view commands 
            1. List Students in a Club
                This would be similar to the SQL command 
                    SELECT s.Student_ID, s.Name, s.Major, s.Class
                    FROM Student s
                    JOIN Membership m ON s.Student_ID = m.Student_ID
                    WHERE m.Club_ID = "id"
            2. List Clubs a student is in
                This would be similar to the SQL command
                    SELECT c.Club_ID, c.Club_Name, c.Description
                    FROM Club c
                    JOIN Membership m ON c.Club_ID = m.Club_ID
                    WHERE m.Student_ID = "id"

    Everything should run and work when tested. 
    Running initialy was causing permission issues for some group members but that was resolved when switching the app.py to access files without using "import os"
    However Add.py, Edit.py, Delete.py, and View.py all import os and it seemed to be ok on everyone's devices.
